# WelOPC

WelOPC 是一个围绕 Agent 生态和 OPC 场景包构建的 AI 信号雷达。当前仓库来自线上 `welopc.com` 已部署版本，包含 AI HOT 情报工作台、公众号爆文采集、OPC 一人公司筛选、Agent 场景包页面、后台管理端和部署脚本。

## 目录

- `app/`：后端源码、Vue 前端源码、Agent 场景包源码、后台页面、配置模板与部署脚本。
- `app/src/ecom_research/aihot_service.py`：AI HOT 后端服务入口，负责公开接口、后台接口、公众号动态采集、频道筛选、日报与模型增强。
- `app/frontend-vue/`：AI HOT Vue 前端源码。
- `app/agent-vue/`：Agent 场景包 Vue 前端源码。
- `app/admin-web/`：后台管理端静态页面。
- `web-dist/`：当前服务器线上静态发布产物，可作为部署参考。
- `.env.example`：运行环境变量模板，不包含任何真实密钥。

## 功能模块

WelOPC 当前包含一套完整的 AI 情报采集、筛选、展示和运营后台能力，核心目标是把分散的 AI / Agent / OPC 相关信号整理成可行动的信息流。

- **AI HOT 情报工作台**：提供精选、全部动态、AI 日报、公众号爆文、OPC 一人公司等栏目，面向日常浏览和快速判断。
- **多源内容采集**：支持官方 RSS、GitHub 热门项目、Product Hunt、Reddit、YouTube、技术博客、媒体源、公众号动态源以及自定义 JSON/API 源。
- **公众号爆文采集**：可配置公众号账号，通过授权后的动态采集链路拉取文章，落库后进行标题、摘要、时间、来源、热度和相关性处理。
- **频道化内容分发**：不同栏目不是简单复用同一批文章，而是按栏目目标使用不同筛选规则、权重和数量限制。
- **AI 日报生成**：围绕当日重要 AI 信号生成日报入口，适合沉淀每日变化、产品发布、模型进展、开源项目和行业趋势。
- **OPC 一人公司栏目**：从所有信源中筛选适合一人公司、独立开发者、Agent 自动化、获客、交付、效率工具和轻量商业化的内容。
- **DeepSeek 增强层**：在配置模型密钥后，可对文章进行标题优化、摘要清洗、标签提取、质量评分、重要性判断和入选理由生成。
- **后台管理端**：提供信源健康、文章管理、日报、公众号源、策略、AI 处理、模型评估、流水线、反馈、访问、用户、系统和审计等运营页面。
- **数据清洗与去重**：对标题、摘要、HTML 残留、重复链接、重复主题、异常来源、过短内容和低相关内容做基础清理。
- **前端展示层**：AI HOT 和 Agent 场景包均提供已上线静态发布物，同时保留 Vue 源码，便于继续二开。
- **定时更新链路**：支持通过定时任务周期性采集、导入、评分、生成频道视图并刷新前端接口数据。
- **本地与生产两套形态**：`app/` 用于源码开发和服务部署，`web-dist/` 用于参考当前线上静态发布结果。

## 筛选策略

系统的筛选逻辑分为“统一入库”和“分频道出榜”两层。统一入库负责把不同信源清洗成同一套文章结构；分频道出榜再根据栏目目标做差异化排序和过滤。

- **统一入库策略**：所有来源先归一化为标题、摘要、链接、来源名、发布时间、原始元数据、标签、热度分、AI 相关分和可信度信息。
- **基础相关性判断**：优先保留 AI、Agent、大模型、自动化、开发工具、生产力、开源项目、模型发布、研究论文、商业化和一人公司相关内容。
- **可信度加权**：官方源、一手发布、开源仓库、明确作者和高质量技术媒体会获得更高基础权重；来源不明、标题党、重复转载和弱相关内容会降权。
- **时间衰减**：越新的内容基础权重越高；旧内容仍可入选，但需要有更高的长期价值、教程价值、工具价值或复用价值。
- **热度评分**：结合 GitHub star 增长、社区讨论、来源权重、发布时间、关键词强度、公众号文章权重和人工策略权重生成综合分。
- **去重策略**：按 URL、标题相似度、主题相似度和来源聚类去重，避免同一事件在多个来源重复霸榜。
- **精选栏目**：偏向高可信、高影响、高新鲜度、高可行动性的内容，过滤弱相关、重复、纯营销、摘要不足和低质量转载。
- **全部 AI 动态**：作为统一信息流，覆盖面更广，保留更多来源和主题，但仍会做基础清洗、去重、排序和低质过滤。
- **AI 日报**：按日期聚合重要事件，优先选择模型发布、产品更新、行业变化、开源项目、研究进展和对 OPC 有启发的内容。
- **公众号爆文**：只从公众号动态池和配置的公众号源中出榜，重点看账号权重、文章新鲜度、AI 相关度、标题信息密度、摘要质量和传播潜力。
- **OPC 一人公司**：面向个人创业者和小团队，优先选择 Agent 自动化、低成本工具、获客增长、内容生产、交付效率、商业模式、独立开发和可落地 SOP。
- **模型增强策略**：配置 DeepSeek 后，模型结果作为编辑层补充，不直接替代基础规则；最终排序仍综合规则分、模型分、来源分和频道策略。
- **后台策略调节**：后台可查看信源健康、模型处理结果、频道视图和流水线状态，后续可以继续把栏目权重、关键词和阈值做成可配置策略。

## 已移除内容

为了安全开源，仓库不包含以下线上运行态内容：

- 管理员密码、服务器凭据和运行时 `.env`。
- DeepSeek、GitHub、公众号授权 Cookie、Token 或任何 API Key。
- SQLite 数据库、WAL/SHM 文件、备份包和日志。
- Python 虚拟环境、Node 依赖目录和缓存文件。

## 本地运行后端

```bash
cd app
python -m venv .venv
source .venv/bin/activate
python -m pip install -U pip
python -m pip install -e .
cp ../.env.example .env
python -m ecom_research.aihot_service init
python -m ecom_research.aihot_service serve --host 127.0.0.1 --port 8790
```

Windows PowerShell 可将激活命令换成：

```powershell
.\.venv\Scripts\Activate.ps1
```

## 本地运行 AI HOT 前端

```bash
cd app/frontend-vue
npm install
npm run dev
```

## 本地运行 Agent 场景包前端

```bash
cd app/agent-vue
npm install
npm run dev
```

## 生产部署参考

线上版本使用 Python 后端服务加静态前端发布：

- 后端服务监听 `127.0.0.1:8790`。
- Web 根目录可参考 `web-dist/`。
- 反向代理将 `/ai-hot/api/*` 转发到后端服务。
- 定时任务每日运行 `python -m ecom_research.aihot_service import-latest` 更新内容。

部署前请复制 `.env.example` 为 `.env` 并填入自己的运行路径和模型密钥。
